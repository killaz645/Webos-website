Step 05

Upload Your Website

With your site created, you're ready to upload your website files and go live. There are two main ways to do this — a web file manager and SFTP. This step takes around 15 minutes.

1

Choosing an Upload Method

You have two main options for getting your files onto the server:

  • Web File Manager — good for small uploads or quick edits directly in the browser
  • SFTP client — better for larger sites, drag-and-drop uploads, and ongoing file management
Tip
For most projects, using an SFTP client like FileZilla is the most comfortable long-term workflow. It feels like copying files on your own computer.
2

Upload via Web File Manager

In your server's file manager, click Files (or open the file manager in your hosting control panel). Navigate to your site's root directory:

/www/wwwroot/your-domain.com

Click the Upload button in the toolbar. You can upload individual files or a .zip archive. If you upload a zip, right-click it afterward and select Decompress to extract it in place.

Warning
Make sure to delete or replace the default index.html placeholder file that some control panels create when you set up the site, otherwise it will appear instead of your website.
3

Upload via SFTP

Download and install FileZilla (free) on your main computer. Open it and connect using the following settings:

  • Host: your server's IP address
  • Username: your Ubuntu server username
  • Password: your Ubuntu server password
  • Port: 22

Once connected, navigate on the right panel to your site directory and drag your local files across:

/www/wwwroot/your-domain.com/
Tip
You can also use the command-line scp tool to copy files to your server if you prefer not to install a GUI application: 
scp -r ./my-website/* username@your-server-ip:/www/wwwroot/your-domain.com/
4

Fix File Permissions

After uploading, make sure the web server can read your files. SSH into your server and run:

sudo chown -R www:www /www/wwwroot/your-domain.com
sudo find /www/wwwroot/your-domain.com -type f -exec chmod 644 {} \;
sudo find /www/wwwroot/your-domain.com -type d -exec chmod 755 {} \;

This sets the correct owner and permissions so Nginx can serve your files properly.

5

Test Your Site

Open a browser and navigate to your server's IP address or domain name. You should see your website load.

If you see a blank page or an error, check the Nginx error log in your control panel under the site's error log, or via the terminal:

sudo tail -f /www/wwwlogs/your-domain.com.error.log
Tip
Common issues are wrong file permissions, a missing index.html in the root directory, or a typo in the domain name configuration.
6

Enable HTTPS with a Free SSL Certificate

To serve your site over HTTPS, many control panels provide Let's Encrypt integration. Alternatively, you can use Certbot to obtain a certificate from Let's Encrypt via the command line.

Using Certbot with Nginx typically looks like this:

sudo apt update
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
Warning
Let's Encrypt only works with a real domain name pointed at your server's public IP. It won't work with a bare IP address or a local network address.
Tip
Many control panels automatically renew Let's Encrypt certificates before they expire. If using Certbot, automatic renewal is configured by a systemd timer.

You're live!

Your website is now running on your own Ubuntu Server. You've completed the full setup guide.

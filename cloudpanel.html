CloudPanel

Install CloudPanel on Ubuntu Server 24.04

CloudPanel provides a lightweight hosting control panel for managing websites, databases, and SSL. Follow these steps to install the Community Edition on a fresh Ubuntu Server 24.04 LTS machine.

1

Overview

CloudPanel installs and manages Nginx, PHP, and MariaDB (or MySQL) and exposes a web UI on port 8443. The installer automates dependencies and performs recommended optimizations for production use.

2

Prerequisites

On a fresh Ubuntu Server 24.04 install, ensure the system is updated and has a reachable IP or domain. Recommended minimums: 1 vCPU, 1 GB RAM (2 GB+ recommended for production).

sudo apt update
sudo apt upgrade -y
sudo apt install -y curl wget gnupg2 ca-certificates

Optional but recommended: set a sensible hostname and ensure ports 80, 443, and 8443 are available.

sudo hostnamectl set-hostname webos
hostnamectl && ip addr show
3

Install CloudPanel (Community Edition)

Run the official installer script. The script will prompt for confirmation and then install required components.

# fetch and run the CloudPanel installer
curl -sSL https://installer.cloudpanel.io/ce/v2/install.sh | sudo bash

Notes:

  • The installer may ask for a domain or IP to configure TLS — you can use the server IP (for local networks) or a DNS name.
  • Installer sets up Nginx, PHP, MariaDB and the CloudPanel service.
4

Open Firewall Ports

Allow HTTPS and CloudPanel's admin port through UFW (adjust interface as needed):

sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 8443/tcp
sudo ufw reload
sudo ufw status verbose
5

Accessing CloudPanel

When installation completes, open the panel in your browser at:

https://192.168.1.1:8443/

Replace 192.168.1.1 with your server IP or hostname. On first visit you will create the admin user and complete initial setup.

6

Verify Installation

sudo systemctl status cloudpanel
sudo systemctl status nginx
ss -tlnp | grep -E ":(80|443|8443)\b"
sudo journalctl -u cloudpanel -n 100

Confirm the panel is reachable and Nginx is serving sites as expected.

7

Troubleshooting

Common checks if the installer fails or the panel is unreachable:

# Check installer logs
sudo journalctl -u cloudpanel -b

# Validate nginx config
sudo nginx -t

# Check open ports
ss -tlnp | egrep ':(80|443|8443)\b'

# Re-run installer if necessary (it is idempotent in most steps)
curl -sSL https://installer.cloudpanel.io/ce/v2/install.sh | sudo bash

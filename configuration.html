Step 03

Configure Ubuntu Server

After Ubuntu Server is installed and rebooted, prepare it for aaPanel by installing a few useful tools, assigning a static IP, and making sure SSH is ready. This step takes around 15 minutes.

1

Install Useful Tools

Install a few standard utilities you'll use while managing the server:

  • curl and wget for downloading files
  • net-tools for network diagnostics like ifconfig
  • htop for live system monitoring
  • nano and vim for text editing
sudo apt install -y curl wget net-tools htop nano vim
2

Verify Installation

Check that the server is running the expected kernel and architecture:

uname -a

You should see output similar to Linux webserver ... 6.8.0-... #1 SMP ... x86_64 GNU/Linux.

Tip
If the command returns the kernel and architecture you expect, the base install is healthy and ready for network configuration.
3

Find Your Network Interface

List your interfaces so you know which one to configure with a static IP:

ip link show

Look for names like eth0, ens0, or ens3. Do not use lo, which is the localhost interface.

Tip
Use the interface name shown here in the Netplan config in the next step.
4

Configure a Static IP Address

Create or edit the Netplan configuration file:

sudo nano /etc/netplan/99-custom-net.yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    ens3:
      dhcp4: no
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

192.168.1.100/24 is your static IP, 192.168.1.1 is your gateway or router IP, and Google DNS is used for name resolution. Replace ens3 with the interface name from the previous step if yours is different.

Apply the configuration and verify that the address is assigned:

sudo netplan apply
ip addr show ens3
ping -c 4 8.8.8.8
Warning
A wrong Netplan file can disconnect your network. Keep local access available while you test the new configuration.
5

Enable and Verify SSH

Make sure SSH is running so you can manage the server remotely:

sudo systemctl enable --now ssh
sudo systemctl status ssh

You want to see active (running) in the service status. If it is not active, start it again and check for errors before moving on.

Tip
Once SSH is working, you can disconnect the monitor and continue administering the server from your main computer.

Step 03

Install aaPanel

aaPanel is a free, open-source web hosting control panel. It gives you a graphical interface to manage your web server, databases, FTP, and more — without needing to configure everything manually. This step takes around 20 minutes.

1

What is aaPanel?

aaPanel (also known as aapanel.com) is a lightweight server control panel that installs Nginx or Apache, PHP, MySQL, and other components in one click. It runs on your server and is accessed through a web browser from any device on your network.

Tip
aaPanel is the international version of BaoTa Panel (宝塔面板), a widely-used Chinese hosting panel. It's actively maintained and works great on Ubuntu Server.
2

Run the Installer

SSH into your server and run the official aaPanel installation script. The script will install all dependencies automatically:

wget -O install.sh https://www.aapanel.com/script/install_6.0_en.sh
sudo bash install.sh aapanel

The installation will take several minutes. When it finishes, it will print your panel URL, username, and a randomly generated password. Save these immediately.

Warning
Do not close your terminal until you have copied your panel login credentials. The password is only shown once during installation.
3

Access the Panel

Open a browser on any computer on your network and navigate to the URL shown after installation. It will look something like:

http://your-server-ip:7800/xxxxxxxxxxxx

Log in with the username and password provided by the installer. You'll be prompted to install a web stack on first login.

Tip
If you can't reach the panel, make sure port 7800 is open in your server's firewall. See the next step for firewall configuration.
4

Install the Web Stack

After logging in, aaPanel will ask you to choose a web stack. For most use cases, select the LNMP option (Linux, Nginx, MySQL, PHP) and click One-click install.

This installs everything you need to host websites. The installation runs in the background and takes 5–10 minutes. You can monitor progress in the task log at the bottom of the screen.

Tip
Nginx is recommended over Apache for most modern websites — it uses less memory and handles high traffic better.
5

Configure the Firewall

In the aaPanel dashboard, go to Security and make sure the following ports are open:

  • 80 — HTTP web traffic
  • 443 — HTTPS web traffic
  • 7800 — aaPanel interface
  • 22 — SSH access

You can also open these ports directly on your server using UFW:

sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 7800
sudo ufw allow 22
sudo ufw enable
6

Create Your First Site

In the aaPanel sidebar, go to Website and click Add site. Fill in your domain name (or your server's IP address for local testing), select your PHP version, and click Submit.

aaPanel will create the site directory, configure Nginx, and set up a database if needed. Your site root folder will be at:

/www/wwwroot/your-domain.com
Tip
If you're using a real domain name, point your domain's DNS A record to your server's public IP address before creating the site in aaPanel.

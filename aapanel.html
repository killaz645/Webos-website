Step 04

Install aaPanel

aaPanel is a free, open-source web hosting control panel. It gives you a graphical interface to manage your web server, databases, FTP, and more — without needing to configure everything manually. This step takes around 20 minutes.

1

What is aaPanel?

aaPanel (also known as aapanel.com) is a lightweight server control panel that installs Nginx or Apache, PHP, MySQL, and other components in one click. It runs on your server and is accessed through a web browser from any device on your network.

Tip
aaPanel is the international version of BaoTa Panel (宝塔面板), a widely-used Chinese hosting panel. It's actively maintained and works great on Ubuntu Server.
2

SSH Into Your Server, Then Run the Installer

First connect to the Ubuntu server from your main computer using the server's username and IP:

ssh webserver@192.168.1.100

Once you are logged in, download and run the official aaPanel installer. It will install all dependencies automatically:

wget -O install.sh https://www.aapanel.com/script/install_6.0_en.sh
sudo bash install.sh aapanel

The installation will take several minutes. When it finishes, it will print your panel URL, username, and a randomly generated password. Save those credentials immediately.

Warning
Do not close your terminal until you have copied your panel login credentials. The password is only shown once during installation.
3

Access the Panel

Open a browser on any computer on your network and use the aaPanel address shown at the end of the installation. The terminal will display both the internal and external login URLs, along with the temporary username and password.

Use those credentials to log in to aaPanel for the first time. After that, the dashboard will prompt you to choose a web stack.

Tip
If you cannot reach the panel, make sure port 7800 is open in your server's firewall. See the firewall step later on this page.

The installer prints the panel URLs and temporary login credentials here.

Use the temporary credentials to sign in to the aaPanel dashboard.

4

Install the Web Stack

aaPanel will ask you to choose a software package set. For this project, choose the Single WebServer Model.

Choose This
Single WebServer Model is the right fit here because you have one physical machine, you only need one website stack, and it keeps the setup simple enough to explain clearly.

Why this is the right choice:

  • You have one physical machine, not a distributed server cluster.
  • It is easier to configure, demonstrate, and troubleshoot.
  • It fits the scope of a single website hosted on one server.
  • It is better for learning because each component is visible in one place.
  • It is simpler to present to evaluators because the architecture is straightforward.

What it does:

  • One Nginx or Apache instance handles all virtual hosts.
  • You can still host multiple websites, but they run on the same web server process.
  • All services remain on the same machine, which keeps the demo easy to follow.

Skip this: the Multi-WebServer Model is designed for larger distributed deployments with load balancing and multiple physical servers. It adds complexity you do not need for this project.

After selecting the Single WebServer Model, continue with the one-click install. The packages will install in the background and you can watch progress in the task log.

Tip
The Single WebServer Model keeps the stack focused on one machine, which makes the rest of the guide easier to follow.

Select the Single WebServer Model on the package selection screen.

Watch the task queue while aaPanel installs the web stack components.

5

Configure the Firewall

In the aaPanel dashboard, go to Security and make sure the following ports are open:

  • 80 — HTTP web traffic
  • 443 — HTTPS web traffic
  • 7800 — aaPanel interface
  • 22 — SSH access
  • samba — file sharing for later use

You can also open these ports directly on your server using UFW:

sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 7800
sudo ufw allow 22
sudo ufw allow samba
sudo ufw enable
6

Create Your First Site

In the aaPanel sidebar, go to Website and click Add site. Fill in your domain name (or your server's IP address for local testing), select your PHP version, and click Submit.

aaPanel will create the site directory, configure Nginx, and set up a database if needed. Your site root folder will be at:

/www/wwwroot/your-domain.com
Tip
If you're using a real domain name, point your domain's DNS A record to your server's public IP address before creating the site in aaPanel.
